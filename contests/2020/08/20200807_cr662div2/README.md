## C問題

シミュレーションは、残っている数の大きいものから優先的に使っていくのが貪欲で賢い。
自分はpriority queueを使った。

また、使えないものを別のスライスにおいておくという手法でシミュレーションを行ったが、
この部分は、直近 `x` 回で使ったものをmapで管理しておくのが簡単だったかもしれない。
（こうすれば、最後のセクションの処理部分が楽になる気がする。）
